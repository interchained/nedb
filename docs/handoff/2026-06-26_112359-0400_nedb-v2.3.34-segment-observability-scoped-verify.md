# NEDB — Working Spec · v2.3.34

**Unit:** Segment observability + scoped integrity check (`ideas.md` #3)  
**Why this one first:** #1 (flip `--dag-v3` default) and #2 (scheduled compaction) both consume
visibility into pack health (live/dead bytes, compaction pressure) and a cheap integrity check.
#3 is the sensor layer they depend on. Ship the sensor before the actuator.  
**Final spec ruling:** `SegmentStore` reports physical packed-storage facts. `Db` computes live/dead truth
from the current id-index. Do **not** treat `SegmentStore.index.len()` as live-object count.  
**Status:** finalized · implementation-ready · zero behavior change in v2-loose mode.

-----

### 1.2 `PhysicalSegmentStats` (new, `segment.rs`)

`SegmentStore` cannot know which hashes are live because it does not own the document id-index.
It can only report the physical segment-pack state.

```rust
const FRAME_BYTES: u64 = 4; // u32 content_len prefix per segment record

#[derive(Clone, Copy, Debug, Default, serde::Serialize)]
pub struct PhysicalSegmentStats {
    pub segment_count:        usize, // count of seg-*.dat
    pub stored_objects:       usize, // SegmentStore.index.len(); all segment-addressable hashes
    pub stored_content_bytes: u64,   // Σ SegmentLocation.len over the segment index
    pub total_bytes:          u64,   // Σ file size of seg-*.dat on disk
    pub framing_bytes:        u64,   // stored_objects × FRAME_BYTES
    pub active_segment_id:    u32,
    pub active_segment_bytes: u64,   // active.offset
    pub last_compaction:      Option<CompactStats>,
}
```

**Important:** `stored_objects` is not `live_objects`. It includes current hashes plus superseded hashes
until a compaction pass rewrites the current live set and removes old segments.

### 1.3 `SegmentStats` (new, `segment.rs` or `db.rs`)

`SegmentStats` is the operator-facing v3 health view. It combines physical segment facts with the
current live hash set supplied by `Db`.

```rust
#[derive(Clone, Copy, Debug, Default, serde::Serialize)]
pub struct SegmentStats {
    pub segment_count:        usize,
    pub stored_objects:       usize, // all hashes in segment index
    pub live_objects:         usize, // current hashes from Db id-index
    pub dead_objects:         usize, // stored_objects.saturating_sub(live_objects)
    pub stored_content_bytes: u64,   // all segment-indexed content bytes
    pub live_bytes:           u64,   // content bytes for live hashes only
    pub total_bytes:          u64,   // segment .dat bytes on disk
    pub framing_bytes:        u64,   // stored_objects × FRAME_BYTES
    pub dead_bytes:           u64,   // total_bytes − (live_bytes + live_objects × FRAME_BYTES), floored at 0
    pub active_segment_id:    u32,
    pub active_segment_bytes: u64,
    pub last_compaction:      Option<CompactStats>,
}
```

### 1.4 `SegmentStore::physical_stats(&self) -> PhysicalSegmentStats`

Pure read:

- Walk `self.index` for `stored_objects` and `stored_content_bytes`.
- Lock/read `active` for `active_segment_id` and `active_segment_bytes`.
- `read_dir(self.dir)` + `metadata().len()` for `segment_count` and `total_bytes`.
- Read `last_compaction` from `Mutex<Option<CompactStats>>`.
- No re-hash.
- No mutation.
- O(segments + stored_objects).

Add to `SegmentStore`:

```rust
last_compaction: Mutex<Option<CompactStats>>,
```

Initialize to `None` in `open_with_max()`. Set it at the tail of `compact()` after `CompactStats`
is computed and before returning.

### 1.5 `SegmentStore::stats(&self, live: &HashSet<String>) -> SegmentStats`

Final API shape for accurate live/dead accounting:

```rust
pub fn stats(&self, live: &HashSet<String>) -> SegmentStats
```

Algorithm:

1. Get `physical = self.physical_stats()`.
2. Iterate `live`; for each hash present in `self.index`, add its `SegmentLocation.len` into `live_bytes`.
3. `live_objects = count(live hashes present in segment index)`.
4. `dead_objects = physical.stored_objects.saturating_sub(live_objects)`.
5. `dead_bytes = physical.total_bytes.saturating_sub(live_bytes + live_objects as u64 * FRAME_BYTES)`.
6. Copy active/compaction fields from `physical`.

This keeps layering correct:

- `Db` decides what is live.
- `SegmentStore` measures where bytes live.

### 1.6 `ObjectStore::segment_stats(&self, live: &HashSet<String>) -> Option<SegmentStats>`

### 1.7 `Db::current_live_hashes(&self) -> HashSet<String>`

Factor out the same live-set walk already used by `Db::compact()`.

```rust
fn current_live_hashes(&self) -> std::collections::HashSet<String> {
    let mut live = std::collections::HashSet::new();
    for coll in self.id_index.collections() {
        for id in self.id_index.list_ids(&coll) {
            if let Some(h) = self.id_index.get(&coll, &id) {
                live.insert(h);
            }
        }
    }
    live
}
```

Then update `compact()` to use this helper.

### 1.8 `Db::segment_stats(&self) -> Option<SegmentStats>`

```rust
pub fn segment_stats(&self) -> Option<crate::segment::SegmentStats> {
    let live = self.current_live_hashes();
    self.objects.segment_stats(&live)
}
```

This is the canonical operator view.

### 1.9 Scoped verify (fast path)

```rust
// segment.rs
pub fn verify_segment(&self, id: u32) -> (usize, Vec<String>)

// store.rs
pub fn verify_segment(&self, id: u32) -> Option<(usize, Vec<String>)>

// db.rs
pub fn verify_segment(&self, id: u32) -> (usize, Vec<String>)
```

`SegmentStore::verify_segment(id)`:

- Iterate only entries where `SegmentLocation.segment_id == id`.
- For each `(hash, loc)`, read `loc.offset..loc.offset+loc.len`.
- Recompute BLAKE2b over content.
- Compare recomputed hash to stored hash key.
- Return `(checked_count, bad_hashes)`.
- Do not mutate state.
- Do not full-scan unrelated segments.
- Missing segment or empty segment returns `(0, vec![])`.

`ObjectStore::verify_segment(id)` returns `None` in v2-loose mode.  
`Db::verify_segment(id)` returns `(0, vec![])` in v2-loose mode so HTTP stays 200-compatible.

Whole-store path remains `verify_all()` / `Db::verify()`.

### 2.2 `GET /v1/databases/:name/verify?segment=N`

Patch the existing verify handler; do not create a second route.

- Without `?segment=N` → existing full `Db::verify()`.
- With `?segment=N` → scoped `Db::verify_segment(N)`.

Do **not** remove existing response fields. Current clients may depend on them.

Recommended response:

```jsonc
{
  "ok": true,
  "seq": 123,
  "head": "b2...",
  "tamper_evident": true,
  "scope": "segment",
  "segment": 1,
  "objects_checked": 50000,
  "tampered": [],

  // optional aliases for operator ergonomics
  "checked": 50000,
  "bad": []
}
```

For full verify, keep `scope: "full"` if adding scope.

Register `GET /v1/databases/:name/segments` in `router()` beside:

```rust
.route("/v1/databases/:name/verify", get(verify_database))
.route("/v1/databases/:name/checkpoint", post(checkpoint))
```

## 8. Final implementation note

The central invariant for this release:

> Segment storage knows physical bytes. Database indexes know live truth. Any spec, metric, or endpoint that calls `SegmentStore.index.len()` “live objects” is wrong.
